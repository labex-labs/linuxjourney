## Lesson Content

Команда `tr`, сокращение от translate (перевод), — это утилита командной строки, которая переводит, удаляет или сжимает символы из стандартного ввода. Она полезна для простой обработки текста и часто используется вместе с конвейерами для обработки вывода других команд.

Основной синтаксис:

```bash
tr [OPTIONS] SET1 [SET2]
```

В отличие от таких инструментов, как `sed` или `awk`, `tr` работает посимвольно. Она не понимает слова, столбцы или регулярные выражения так же, как они. Это делает её быстрой и простой для задач, таких как изменение регистра, удаление цифр и нормализация повторяющихся пробелов.

### Основной перевод символов

Самое распространённое использование `tr` — это замена одного набора символов на другой. Например, вы можете легко перевести все строчные буквы в заглавные.

```bash
$ echo "hello world" | tr a-z A-Z
HELLO WORLD
```

В этом примере мы передали вывод команды `echo` в команду `tr`. Команда `tr` затем перевела диапазон символов `a-z` в соответствующие символы диапазона `A-Z`.

Вы также можете заменить один символ на другой:

```bash
$ echo "2026-06-23" | tr '-' '/'
2026/06/23
```

Каждый символ из `SET1` сопоставляется с символом на той же позиции в `SET2`.

```bash
$ echo "abc123" | tr 'abc' 'ABC'
ABC123
```

Здесь `a` становится `A`, `b` становится `B`, а `c` становится `C`.

### Удаление символов с помощью -d

Ещё одна мощная функция — возможность удалять определённые символы с помощью опции `-d`. Это особенно полезно для очистки текста. Например, если вы хотите удалить все цифры из строки, используйте:

```bash
$ echo "My address is 123 Main Street" | tr -d '0-9'
My address is  Main Street
```

Здесь `tr -d` удалил каждый символ из указанного набора, от `0` до `9`.

Вы можете удалить знаки препинания из строки, используя класс символов:

```bash
$ echo "Hello, world!" | tr -d '[:punct:]'
Hello world
```

Также можно удалить символы новой строки, чтобы объединить строки:

```bash
$ printf "one\ntwo\nthree\n" | tr -d '\n'
onetwothree
```

### Сжатие повторяющихся символов

Команда `tr` также может "сжимать" повторяющиеся символы в один экземпляр с помощью опции `-s`. Это отлично подходит для нормализации текста с лишними пробелами.

```bash
$ echo "Hello      World,   how   are   you?" | tr -s ' '
Hello World, how are you?
```

В этом случае `tr -s ' '` заменил последовательности из нескольких пробелов одним пробелом, делая вывод гораздо чище.

Вы также можете сжимать повторяющиеся символы новой строки:

```bash
$ printf "one\n\n\nTwo\n" | tr -s '\n'
one
Two
```

### Использование классов символов

Классы символов делают команды `tr` более читаемыми и переносимыми. Распространённые классы включают:

- `[:lower:]`: строчные буквы.
- `[:upper:]`: заглавные буквы.
- `[:digit:]`: цифры.
- `[:alpha:]`: буквы.
- `[:alnum:]`: буквы и цифры.
- `[:space:]`: пробельные символы.
- `[:punct:]`: знаки препинания.

Например, преобразуйте строчный текст в заглавный с помощью классов символов:

```bash
$ echo "linux journey" | tr '[:lower:]' '[:upper:]'
LINUX JOURNEY
```

Удалите всё, кроме букв и цифр, используя `-c` вместе с `-d`. Опция `-c` означает дополнение, или «всё, что не входит в этот набор».

```bash
$ echo "user@example.com!" | tr -cd '[:alnum:]'
userexamplecom
```

### Комбинирование удаления и сжатия

Вы можете комбинировать опции при очистке текста. В этом примере удаляются знаки препинания, затем сжимаются повторяющиеся пробелы:

```bash
$ echo "Hello,,,     world!!!" | tr -d '[:punct:]' | tr -s ' '
Hello world
```

Для табуляции входных данных вы можете заменить табуляции на запятые:

```bash
$ printf "name\tlevel\npete\tbeginner\n" | tr '\t' ','
name,level
pete,beginner
```

### Распространённые опции tr

Вот опции, которые вы будете использовать чаще всего:

- `-d`: удалять символы из `SET1`.
- `-s`: сжимать повторяющиеся символы из `SET1`.
- `-c`: использовать дополнение к `SET1`.
- `-t`: усечь `SET1` до длины `SET2` перед переводом.

### Часто задаваемые вопросы

**Почему tr читает из конвейера?** `tr` читает из стандартного ввода, поэтому её часто используют после команд, таких как `echo`, `cat`, `printf` или других команд, которые выводят текст.

**Заменяет ли tr слова?** Нет. `tr` переводит символы, а не слова. Используйте `sed`, если нужно заменить целые слова или шаблоны.

**Почему моя команда tr меняет символы по одному?** Так работает `tr`. Она сопоставляет каждый символ из `SET1` с соответствующим символом из `SET2`.

**Почему нужно заключать наборы в кавычки, например 'a-z'?** Кавычки предотвращают интерпретацию оболочкой специальных символов до того, как они попадут в `tr`.
