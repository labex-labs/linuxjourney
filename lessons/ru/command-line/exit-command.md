## Lesson Content

Поздравляем с завершением базовых уроков вашего путешествия в Linux! Вы изучили основные основы Linux, и теперь пришло время научиться правильно завершать сеанс. Выход из оболочки Linux — это простой, но важный завершающий шаг.

### Команда Exit

Самый распространённый способ завершить сеанс оболочки — использовать команду `exit`. Когда вы вводите `exit` и нажимаете Enter, текущий процесс оболочки завершается. Эта команда работает практически в любой среде оболочки.

```bash
$ exit
```

Если вы открыли окно терминала, команда `exit` обычно закрывает оболочку, запущенную внутри него. Если вы подключились через SSH, `exit` завершает удалённый сеанс оболочки и возвращает вас к локальному приглашению.

### Коды Возврата Exit

Команда `exit` также может возвращать код состояния. Код `0` обычно означает успешное выполнение, а ненулевой код — ошибку или особое состояние.

```bash
$ exit 0
```

Вы будете чаще сталкиваться с кодами возврата при написании скриптов оболочки. Для интерактивного использования достаточно просто ввести `exit`.

### Команда Logout

Ещё одна команда, которую можно использовать для выхода из терминала — это `logout`. Эта команда специально предназначена для завершения сеанса входа в систему (login shell). Хотя во многих современных системах она ведёт себя похоже на `exit`, полезно знать обе команды.

```bash
$ logout
```

### Закрытие Окна Терминала

Если вы работаете в графическом интерфейсе, у вас также есть возможность просто закрыть окно терминала. Это действие обычно посылает сигнал, который завершает сеанс оболочки, запущенный внутри.

### Распространённые Способы Выйти из Оболочки

- Введите `exit`, чтобы завершить текущую оболочку.
- Нажмите `Ctrl-D` на пустой строке приглашения, чтобы послать сигнал конца файла, что часто приводит к выходу из оболочки.
- Введите `logout`, когда вы находитесь в сеансе входа.
- Закройте окно терминала при использовании графического терминала.

### Часто Задаваемые Вопросы

**Является ли exit тем же, что и закрытие окна терминала?** Часто результат похож, но `exit` аккуратно сообщает оболочке о завершении.

**Что такое Ctrl-D?** Это сигнал конца файла для оболочки. На пустой строке приглашения обычно приводит к выходу.

**Что означает exit 1?** Это выход с кодом состояния `1`, обычно используемый для обозначения ошибки в скриптах.
