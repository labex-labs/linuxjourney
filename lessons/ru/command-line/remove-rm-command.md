## Lesson Content

В Linux часто накапливаются файлы, которые больше не нужны. Чтобы удалить их, используется команда `rm` (remove) — базовая утилита для управления вашей файловой системой. Основной синтаксис:

```bash
rm [OPTIONS] FILE...
```

Команда `rm` удаляет записи каталогов из файловой системы. Проще говоря, она удаляет файлы. В отличие от многих графических окружений, удаление через командную строку обычно не перемещает файлы в корзину, поэтому перед нажатием Enter следует внимательно проверить команду.

### Удаление одного файла

Чтобы удалить один файл, передайте имя файла команде `rm`.

```bash
$ rm file1
```

Можно удалить несколько файлов сразу, перечислив их подряд.

```bash
$ rm notes.txt old-report.txt draft.md
```

Это удобно для быстрой очистки, но также означает, что опечатка может привести к удалению большего количества файлов, чем вы планировали.

### Удаление файлов с помощью подстановочных символов

Подстановочные символы оболочки позволяют сопоставлять несколько файлов. Например, эта команда удалит все файлы с расширением `.tmp` в текущем каталоге:

```bash
$ rm *.tmp
```

Перед использованием `rm` с подстановочным символом безопаснее предварительно проверить совпадения с помощью `ls`.

```bash
$ ls *.tmp
cache.tmp  test.tmp
$ rm *.tmp
```

Помните, что оболочка расширяет `*.tmp` до списка файлов до запуска `rm`. Если шаблон совпадает с большим количеством файлов, `rm` получит их все.

### Интерактивное удаление с -i

Для более безопасного подхода используйте опцию `-i`. Она запрашивает подтверждение перед удалением каждого файла.

```bash
$ rm -i important.txt
rm: remove regular file 'important.txt'? y
```

Используйте `rm -i` при удалении файлов из общего каталога, при очистке большого количества файлов или при первом знакомстве с командой.

### Принудительное удаление с -f

Опция `-f` означает «force» (принудительно). Она игнорирует несуществующие файлы и не запрашивает подтверждения.

```bash
$ rm -f old-cache.txt
```

Это полезно в скриптах, где очистка должна продолжаться, даже если файл уже отсутствует.

```bash
$ rm -f build.log
```

Будьте осторожны: `-f` также подавляет некоторые предупреждения, поэтому может скрыть ошибки.

### Удаление каталогов с -r

По умолчанию `rm` не может удалить каталог.

```bash
$ rm projects
rm: cannot remove 'projects': Is a directory
```

Чтобы удалить каталог и всё его содержимое, используйте `-r` или `-R` для рекурсивного удаления.

```bash
$ rm -r old-project
```

Рекурсивное удаление проходит по дереву каталогов и удаляет файлы, подкаталоги и их содержимое.

### Опасности rm -rf

Команда `rm -rf` сочетает рекурсивное удаление с принудительным.

```bash
$ rm -rf old-project
```

Эта команда может быть уместна для удаления сгенерированных папок, например, результатов сборки, но опасна, так как удаляет целое дерево без вопросов. Всегда проверяйте:

- Находитесь ли вы в том каталоге, в котором думаете? Используйте `pwd`.
- Правильно ли расширился ваш подстановочный символ? Проверьте с помощью `ls`.
- Абсолютный или относительный путь? `/tmp/cache` и `tmp/cache` — очень разные пути.
- Нет ли случайного пробела? `rm -rf old-project` и `rm -rf old project` удаляют разные пути.

### Использование rmdir для пустых каталогов

В качестве более безопасной альтернативы удаляйте пустой каталог с помощью `rmdir`.

```bash
$ rmdir empty-directory
```

Команда `rmdir` сработает только если каталог полностью пуст, что делает её более безопасным выбором, чем `rm -r` для задач очистки.

### Часто используемые опции rm

Вот опции, которые вы встретите чаще всего:

- `-i`: Запрашивать подтверждение перед каждым удалением.
- `-I`: Запрашивать подтверждение один раз перед удалением более трёх файлов или рекурсивным удалением.
- `-f`: Принудительное удаление и игнорирование несуществующих файлов.
- `-r` или `-R`: Рекурсивное удаление каталогов и их содержимого.
- `-v`: Показывать, что было удалено.

Например, можно комбинировать опции:

```bash
$ rm -rv old-project
removed 'old-project/notes.txt'
removed directory 'old-project'
```

### Частые вопросы

**Можно ли отменить rm?** Обычно нет. После удаления файла с помощью `rm` встроенной команды отмены нет. Резервные копии, системы контроля версий и инструменты восстановления файловой системы — вот настоящая страховка.

**Почему rm пишет «Permission denied»?** У вас нет прав на удаление этого файла или на изменение каталога, в котором он находится. Проверьте владельца и права с помощью `ls -l`.

**Почему rm пишет «No such file or directory»?** Файл не существует по указанному пути или вы находитесь в другом каталоге, чем ожидали. Используйте `pwd` и `ls` для проверки.

**Стоит ли использовать sudo с rm?** Только если вы полностью понимаете, что удаляете. `sudo rm -r` может удалить системные файлы, к которым обычный пользователь не имеет доступа.
