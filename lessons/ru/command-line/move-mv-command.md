## Lesson Content

Команда `mv`, сокращение от «move» (переместить), является базовой утилитой в любой Linux-среде. Она выполняет две основные функции: переименование файлов или каталогов и перемещение их в другое место.

Основной синтаксис:

```bash
mv [OPTIONS] SOURCE DESTINATION
```

В отличие от `cp`, которая создает копию, `mv` изменяет местоположение исходного объекта или его имя.

### Переименование файлов и каталогов

Одно из самых распространенных применений `mv` — переименование. Синтаксис простой: укажите старое имя и новое имя.

Чтобы переименовать файл:

```bash
$ mv oldfile newfile
```

Та же логика применяется для переименования каталогов:

```bash
$ mv old_directory_name new_directory_name
```

### Перемещение файлов и каталогов

Другая основная функция команды `mv` — перемещение объектов из одного места в другое.

Чтобы переместить один файл в другой каталог:

```bash
$ mv file2 /home/pete/Documents
```

Вы также можете перемещать несколько файлов одновременно. Просто перечислите все исходные файлы, а затем целевой каталог:

```bash
$ mv file_1 file_2 somedirectory/
```

В системах GNU/Linux полезной опцией является `-t`, которая позволяет сначала указать целевой каталог. Это может быть удобнее при перемещении большого количества файлов.

```bash
$ mv -t somedirectory/ file_1 file_2
```

В отличие от команды `cp`, для перемещения каталога не требуется рекурсивная опция. `mv` по умолчанию работает с каталогами.

### Важные опции для команды mv

По умолчанию, если вы перемещаете файл в место, где уже существует файл с таким же именем, `mv` перезапишет его без предупреждения. Чтобы избежать случайной потери данных, можно использовать следующие опции:

- **-i (интерактивный режим)**: важная функция безопасности. Перед перезаписью существующего файла будет запрошено подтверждение.

  ```bash
  $ mv -i source_file destination_directory
  ```

- **-b (создание резервной копии)**: если вы хотите перезаписать файл, но сохранить старую версию, эта опция создаст резервную копию файла назначения. Резервная копия обычно получает суффикс тильды (`~`).

  ```bash
  $ mv -b file1 directory_with_file1
  ```

- **-v (подробный режим)**: эта опция заставляет команду `mv` выводить информацию о выполняемых действиях, показывая каждый перемещаемый или переименовываемый файл.

  ```bash
  $ mv -v file1 file2 somedirectory/
  ```

Еще одна полезная опция — `-n`, которая означает «не перезаписывать». Она предотвращает перезапись существующего файла назначения.

```bash
$ mv -n source_file destination_directory
```

### Распространенные примеры использования mv

Переименование файла:

```bash
$ mv draft.txt final.txt
```

Перемещение каталога:

```bash
$ mv project /home/pete/Documents/
```

Перемещение всех текстовых файлов в папку:

```bash
$ mv *.txt notes/
```

Перед перемещением большого количества файлов рекомендуется предварительно проверить совпадения с помощью `ls`.

### Часто задаваемые вопросы

**Команда mv копирует файлы?** Нет. `mv` перемещает или переименовывает исходный объект.

**Может ли mv перезаписывать файлы?** Да. Используйте `mv -i`, чтобы запрашивать подтверждение, или `mv -n`, чтобы избежать перезаписи.

**Нужно ли использовать mv -r для каталогов?** Нет. `mv` перемещает каталоги без `-r`.
