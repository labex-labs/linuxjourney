## Lesson Content

С учетом огромного количества файлов в системе, найти конкретный файл может быть сложно. Команда `find` ищет по деревьям каталогов, используя критерии, такие как имя, тип, размер и время изменения.

### Использование команды find

Основной синтаксис:

```bash
find [PATH] [EXPRESSION]
```

Вы указываете каталог для поиска и критерии того, что ищете.

Например, чтобы найти файл с именем `puppies.jpg` в каталоге `/home` и всех его подкаталогах, используйте:

```bash
$ find /home -name puppies.jpg
```

Поиск по умолчанию рекурсивный, поэтому `find /home` ищет внутри `/home` и всех его подкаталогов.

### Поиск по имени и типу

Одно из самых распространенных применений `find` — поиск по имени файла. Опция `-name` ищет точное совпадение или по шаблонам в стиле оболочки.

```bash
$ find . -name "*.txt"
```

Вы также можете указать тип объекта, который ищете. Для этого используется опция `-type`. Например, если нужно найти каталог, а не файл, используйте `d`.

```bash
$ find /home -type d -name MyFolder
```

В этой команде мы указываем тип `d` для каталога и ищем элемент с именем `MyFolder`. Чтобы искать именно обычные файлы, используйте `-type f`.

### Поиск по размеру и времени

Можно искать по размеру файла:

```bash
$ find . -type f -size +10M
$ find . -type f -size -1k
```

Первая команда находит файлы больше 10 мегабайт. Вторая — файлы меньше 1 килобайта.

Также можно искать по времени изменения:

```bash
$ find . -type f -mtime -7
$ find . -type f -mtime +30
```

`-mtime -7` означает измененные за последние 7 дней. `-mtime +30` — измененные более 30 дней назад.

### Выполнение действий над результатами

По умолчанию `find` выводит пути совпадений. Можно добавить действия, такие как `-print`, `-delete` или `-exec`.

Явно вывести совпадения:

```bash
$ find . -name "*.log" -print
```

Выполнить `ls -l` для каждого совпадения:

```bash
$ find . -name "*.log" -exec ls -l {} \;
```

Заполнитель `{}` заменяется на каждый найденный путь. Экранированная точка с запятой обозначает конец команды.

Будьте осторожны с разрушительными действиями, такими как `-delete`. Сначала выполните тот же поиск без `-delete`, чтобы проверить результаты.

### Распространенные опции find

- `-name PATTERN`: Поиск по имени файла.
- `-iname PATTERN`: Поиск по имени файла без учета регистра.
- `-type f`: Поиск обычных файлов.
- `-type d`: Поиск каталогов.
- `-size +10M`: Поиск файлов больше 10 мегабайт.
- `-mtime -7`: Поиск файлов, измененных за последние 7 дней.
- `-maxdepth N`: Ограничение глубины поиска.

### Часто задаваемые вопросы

**Почему find показывает "Permission denied"?** Ваш пользователь не имеет прав на чтение некоторых каталогов. Ищите в более узком пути или используйте соответствующие привилегии.

**Почему нужно заключать шаблоны вроде "*.txt" в кавычки?** Кавычки предотвращают расширение шаблона оболочкой до того, как его получит `find`.

**Является ли find рекурсивным?** Да. Поиск по подкаталогам происходит по умолчанию.
