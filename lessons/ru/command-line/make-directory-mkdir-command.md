## Lesson Content

При работе с файлами вам потребуется организовывать их в каталоги. Основным инструментом для этой задачи является команда `mkdir`, что означает «создать каталог».

Основной синтаксис:

```bash
mkdir [OPTIONS] DIRECTORY...
```

### Создание одного каталога

Самое простое использование `mkdir` — создать один новый каталог. Если каталог ещё не существует, эта команда создаст его в текущем расположении.

```bash
$ mkdir documents
```

### Создание нескольких каталогов

Вы также можете создать несколько каталогов одновременно, перечислив их имена через пробел. Это эффективный способ быстро создать несколько папок.

```bash
$ mkdir books paintings
```

### Создание вложенных каталогов

Иногда нужно создать каталог и его родительские каталоги одновременно. Опция `-p` идеально подходит для этого. Она предотвращает ошибки, если родительские каталоги отсутствуют.

```bash
$ mkdir -p books/hemingway/favorites
```

Эта команда создаст `books`, `hemingway` и `favorites`, если они ещё не существуют.

### Установка прав доступа к каталогу

Используйте `-m` для установки прав доступа при создании каталога.

```bash
$ mkdir -m 755 public
```

Вы узнаете больше о правах доступа позже, а пока этот пример создаёт каталог, в который владелец может записывать, а остальные могут читать и заходить.

### Распространённые опции mkdir

- `-p`: Создавать родительские каталоги по необходимости.
- `-m MODE`: Установить права доступа для нового каталога.
- `-v`: Выводить сообщение для каждого созданного каталога.

Пример:

```bash
$ mkdir -pv projects/app/src
mkdir: created directory 'projects'
mkdir: created directory 'projects/app'
mkdir: created directory 'projects/app/src'
```

### Часто задаваемые вопросы

**Почему mkdir выводит «File exists»?** Файл или каталог с таким именем уже существует. Используйте `ls`, чтобы проверить.

**Как создать вложенные каталоги?** Используйте `mkdir -p parent/child/grandchild`.

**Может ли mkdir создавать файлы?** Нет. Для создания пустых файлов используйте `touch`.
