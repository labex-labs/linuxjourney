## Lesson Content

Ввод длинных или повторяющихся команд может быть утомительным. Алиас — это ярлык оболочки, который позволяет определить собственное имя для команды или последовательности команд.

### Создание временного алиаса

Чтобы создать временный алиас, который действует только в текущей сессии терминала, просто укажите имя и присвойте ему строку команды.

Например, создайте алиас с именем `ll` для `ls -la`:

```bash
$ alias ll='ls -la'
```

Теперь вместо ввода `ls -la` вы можете просто ввести `ll`, и будет выполнена та же команда. Это простой, но мощный способ настроить вашу оболочку.

### Сделать алиас постоянным

Временный алиас исчезнет после закрытия терминала или перезагрузки системы. Чтобы сделать `command alias in linux` постоянным, нужно добавить его в файл конфигурации вашей оболочки. Для Bash это обычно файл `~/.bashrc`.

1. Откройте файл в текстовом редакторе: `nano ~/.bashrc`
2. Добавьте определение алиаса в файл так же, как вы вводили его в командной строке:

```bash
alias ll='ls -la'
alias update='sudo apt update && sudo apt upgrade'
```

3. Сохраните файл и выйдите из редактора.

Чтобы изменения вступили в силу, нужно либо закрыть и заново открыть терминал, либо заставить оболочку перечитать файл конфигурации с помощью команды `source`:

```bash
$ source ~/.bashrc
```

Теперь ваш алиас будет доступен при каждом запуске новой сессии Bash.

### Удаление алиаса

Если алиас больше не нужен, его можно удалить командой `unalias`. Это удалит алиас из текущей сессии.

```bash
$ unalias ll
```

Чтобы удалить постоянный алиас, нужно также удалить его определение из файла `~/.bashrc`.

### Список существующих алиасов

Выполните `alias` без аргументов, чтобы увидеть список алиасов в текущей оболочке.

```bash
$ alias
alias ll='ls -la'
alias grep='grep --color=auto'
```

Используйте `type`, чтобы узнать, что будет выполнено при вводе команды:

```bash
$ type ll
ll is aliased to 'ls -la'
```

### Полезные примеры алиасов

```bash
$ alias la='ls -la'
$ alias ..='cd ..'
$ alias grep='grep --color=auto'
```

Держите алиасы короткими и предсказуемыми. Избегайте замены опасных команд неожиданным поведением, если вы не уверены.

### Часто задаваемые вопросы

**Работают ли алиасы в скриптах?** Обычно нет, по умолчанию. В скриптах следует использовать полные команды или функции.

**Куда помещать алиасы Bash?** Помещайте их в `~/.bashrc` для интерактивных сессий Bash.

**Что делать, если алиас конфликтует с реальной командой?** В интерактивной оболочке алиас обычно имеет приоритет. Используйте `command имя` или `\имя`, чтобы обойти алиас.
