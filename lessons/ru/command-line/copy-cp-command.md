## Lesson Content

Команда `cp` — это стандартный инструмент для копирования файлов и директорий в Linux. Она создаёт новую копию, оставляя исходный файл на месте. Основной синтаксис:

```bash
cp [OPTIONS] SOURCE DESTINATION
```

Вы можете скопировать один файл в другой файл, один или несколько файлов в директорию или целое дерево директорий с помощью нужной опции.

### Базовое копирование файлов

Чтобы скопировать файл, укажите исходный файл и директорию или путь назначения.

```bash
$ cp mycoolfile /home/pete/Documents/cooldocs
```

В этом примере `mycoolfile` — исходный файл, а `/home/pete/Documents/cooldocs` — директория назначения. Также можно скопировать файл и дать ему новое имя в месте назначения.

```bash
$ cp mycoolfile /home/pete/Documents/mycoolfile_backup
```

Если место назначения — существующая директория, скопированный файл сохраняет своё оригинальное имя. Если место назначения — имя файла, `cp` создаст копию с этим новым именем.

### Копирование нескольких файлов в директорию

Чтобы скопировать несколько файлов в одну директорию, перечислите все исходные файлы, а в конце укажите директорию назначения.

```bash
$ cp report.txt notes.txt summary.txt /home/pete/Documents/
```

Последний аргумент должен быть директорией, если указано более одного исходного файла.

### Использование подстановочных символов для массового копирования

Подстановочные символы — специальные символы, которые помогают выбрать несколько файлов по шаблону, обеспечивая большую гибкость.

- `*`: соответствует любой последовательности символов.
- `?`: соответствует любому одному символу.
- `[]`: соответствует любому одному символу из указанных в скобках.

Например, чтобы скопировать все изображения JPEG из текущего расположения в директорию `Pictures`:

```bash
$ cp *.jpg /home/pete/Pictures
```

Вы можете предварительно просмотреть файлы, которые соответствуют шаблону, перед копированием:

```bash
$ ls *.jpg
beach.jpg  lunch.jpg  profile.jpg
$ cp *.jpg /home/pete/Pictures
```

### Рекурсивное копирование директорий

Если попытаться скопировать директорию с помощью `cp` без опций, будет ошибка. Чтобы скопировать директорию и всё её содержимое, включая поддиректории, нужно использовать флаг `-r` (рекурсивно).

```bash
$ cp -r Pumpkin/ /home/pete/Documents
```

Эта команда копирует директорию `Pumpkin` и всё, что в ней, в вашу директорию `Documents`.

Вы также можете встретить `-R`, который выполняет ту же рекурсивную функцию на типичных Linux-системах:

```bash
$ cp -R website /home/pete/backups/
```

### Обработка перезаписи файлов

По умолчанию `cp` перезаписывает файл в месте назначения, если там уже есть файл с таким же именем. Чтобы избежать случайной потери данных, используйте флаг `-i` (интерактивный), который запрашивает подтверждение перед перезаписью.

```bash
$ cp -i mycoolfile /home/pete/Pictures
cp: overwrite '/home/pete/Pictures/mycoolfile'? n
```

Если же нужно принудительно перезаписать без запросов, используйте опцию `-f`. Это удобно в скриптах, где взаимодействие с пользователем невозможно.

```bash
$ cp -f mycoolfile /home/pete/Pictures
```

Ещё одна полезная опция безопасности — `-n`, что означает «не перезаписывать». Она предотвращает перезапись существующего файла назначения.

```bash
$ cp -n mycoolfile /home/pete/Pictures
```

### Сохранение атрибутов файла с помощью -p

При копировании файла его метаданные, такие как время изменения и владельцы, обычно обновляются. Чтобы сохранить эти исходные атрибуты, используйте опцию `-p`.

Опция `cp -p` особенно полезна для резервного копирования или при переносе файлов, когда важно сохранить временные метки.

```bash
$ cp -p mycoolfile /home/pete/backups/
```

Это копирует `mycoolfile`, сохраняя режим, владельца (где возможно) и временные метки.

### Архивное копирование с -a

Опция `-a` означает архивный режим. Она часто используется для копирования директорий в стиле резервного копирования, так как сохраняет множество атрибутов и копирует рекурсивно.

```bash
$ cp -a project/ project-backup/
```

Для многих повседневных резервных копий `cp -a` удобнее, чем вручную комбинировать несколько опций.

### Копировать только более новые файлы с -u

Опция `-u` копирует только если исходный файл новее файла назначения или если файла назначения не существует.

```bash
$ cp -u *.txt /home/pete/Documents/
```

Это полезно при обновлении папки без перезаписи уже актуальных файлов.

### Часто используемые опции cp

Вот опции, которые вы будете использовать чаще всего:

- `-r` или `-R`: копировать директории рекурсивно.
- `-i`: спрашивать перед перезаписью файла.
- `-f`: принудительно перезаписывать, удаляя файл назначения при необходимости.
- `-n`: не перезаписывать существующие файлы.
- `-p`: сохранять режим, владельца (где возможно) и временные метки.
- `-a`: архивный режим, полезен для сохранения деревьев директорий.
- `-u`: копировать только если исходный файл новее файла назначения.
- `-v`: показывать каждый файл при копировании.

### Часто задаваемые вопросы

**Почему cp перезаписал мой файл?** По умолчанию `cp` заменяет файл назначения с таким же именем. Используйте `cp -i` для запроса подтверждения или `cp -n`, чтобы избежать перезаписи.

**Почему cp не может скопировать директорию?** Для директории требуется рекурсивное копирование. Используйте `cp -r source-dir destination-dir`.

**В чём разница между cp и mv?** `cp` создаёт копию и сохраняет оригинал. `mv` перемещает или переименовывает оригинал.

**Стоит ли использовать cp -r или cp -a для резервного копирования?** Используйте `cp -r` для простого рекурсивного копирования. Используйте `cp -a`, когда нужен резервный копийный стиль с сохранением большего количества атрибутов файлов.
