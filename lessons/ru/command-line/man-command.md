## Lesson Content

В Linux почти каждая команда имеет собственное руководство. Они называются «man-страницы» (сокращение от manual pages) и являются важным ресурсом для эффективного изучения системы.

### Понимание man-страниц

Man-страницы — это встроенная документация для команд Linux, утилит и системных вызовов. Они предоставляют подробное описание того, что делает команда, доступных опций (или флагов) и как её использовать. Это ваш первый и лучший источник помощи в командной строке.

### Доступ к руководству с помощью man

Чтобы просмотреть руководство для любой команды, используйте `man`, за которым следует имя команды. Например, чтобы прочитать руководство по `ls`, введите:

```bash
$ man ls
```

Это откроет man-страницу для `ls`. Вы можете прокручивать с помощью стрелок, искать с помощью `/` и нажать `q` для выхода.

### Поиск информации об опциях команды

Man-страницы особенно полезны для понимания опций команд. Например, если вы видели `ls -l` и хотите узнать, что означает `-l`, откройте `man ls` и найдите `-l`.

Внутри man-страницы:

- Нажмите `/` и введите поисковый запрос для поиска вперёд.
- Нажмите `n` для перехода к следующему совпадению.
- Нажмите `N` для перехода к предыдущему совпадению.
- Нажмите `q` для выхода.

### Понимание разделов man-страниц

Руководства организованы по пронумерованным разделам. Распространённые разделы включают:

- `1`: Пользовательские команды.
- `2`: Системные вызовы.
- `3`: Функции библиотек.
- `5`: Форматы файлов.
- `8`: Команды системного администрирования.

Иногда одно и то же имя встречается в нескольких разделах. Вы можете указать номер раздела:

```bash
$ man 5 passwd
$ man 1 passwd
```

### Часто задаваемые вопросы

**Почему вывод man такой длинный?** Man-страницы — это справочная документация. Используйте поиск внутри `man`, чтобы перейти к нужной части.

**Как выйти из man?** Нажмите `q`.

**Что делать, если man-страница отсутствует?** Попробуйте `COMMAND --help`, `help COMMAND` или установите пакет документации для вашей дистрибуции.
