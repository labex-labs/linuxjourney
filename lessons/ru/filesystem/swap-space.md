## Lesson Content

В нашем предыдущем примере я показал вам, как просмотреть таблицу разделов. Давайте вернемся к этому примеру, а точнее к этой строке:

```
Number  Start   End     Size    Type      File system     Flags
 5      6861MB  7380MB  519MB   logical   linux-swap(v1)
```

Что это за раздел подкачки (swap)? Что ж, подкачка — это то, что мы используем для выделения виртуальной памяти нашей системе. Если у вас не хватает памяти, система использует этот раздел для «выгрузки» (swap) частей памяти неактивных процессов на диск, чтобы вы не испытывали нехватки памяти.

### Использование раздела для пространства подкачки

Предположим, мы хотим настроить наш раздел `/dev/sdb2` для использования в качестве пространства подкачки.

1. Сначала убедитесь, что на разделе ничего нет.
2. Выполните: `mkswap /dev/sdb2` для инициализации областей подкачки.
3. Выполните: `swapon /dev/sdb2`. Это активирует устройство подкачки.
4. Если вы хотите, чтобы раздел подкачки сохранялся после загрузки, вам нужно добавить запись в файл `/etc/fstab`. `sw` — это тип файловой системы, который вы будете использовать.
5. Чтобы удалить подкачку: `swapoff /dev/sdb2`.

Обычно рекомендуется выделять подкачке примерно в два раза больше места, чем у вас оперативной памяти. Однако современные системы сегодня, как правило, достаточно мощные и имеют достаточно ОЗУ.
