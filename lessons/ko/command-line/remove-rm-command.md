## Lesson Content

Linux에서는 더 이상 필요하지 않은 파일이 쌓이는 것이 흔합니다. 이를 삭제하려면 파일 시스템 관리를 위한 기본 유틸리티인 `rm` (remove) 명령어를 사용합니다. 기본 문법은 다음과 같습니다:

```bash
rm [OPTIONS] FILE...
```

`rm` 명령어는 파일 시스템에서 디렉토리 항목을 제거합니다. 쉽게 말해 파일을 삭제하는 것입니다. 많은 데스크탑 환경과 달리, 커맨드 라인에서 삭제는 보통 휴지통으로 이동하지 않으므로, Enter 키를 누르기 전에 명령어를 꼭 확인해야 합니다.

### 단일 파일 삭제

한 개의 파일을 삭제하려면 파일 이름을 `rm`에 전달하면 됩니다.

```bash
$ rm file1
```

여러 파일을 한 번에 삭제하려면 파일 이름을 연속으로 나열할 수 있습니다.

```bash
$ rm notes.txt old-report.txt draft.md
```

빠른 정리에 유용하지만, 오타가 있을 경우 의도보다 많은 파일이 삭제될 수 있으니 주의해야 합니다.

### 와일드카드로 파일 삭제

셸 와일드카드를 사용하면 여러 파일을 한꺼번에 지정할 수 있습니다. 예를 들어, 현재 디렉토리의 모든 `.tmp` 파일을 삭제하려면 다음과 같이 합니다:

```bash
$ rm *.tmp
```

와일드카드와 함께 `rm`을 사용하기 전에는 `ls`로 삭제 대상 파일을 미리 확인하는 것이 안전합니다.

```bash
$ ls *.tmp
cache.tmp  test.tmp
$ rm *.tmp
```

셸이 `rm` 실행 전에 `*.tmp`를 확장한다는 점을 기억하세요. 패턴이 예상보다 많은 파일과 일치하면 `rm`은 모두를 받게 됩니다.

### -i 옵션으로 대화형 삭제

더 안전한 방법으로 `-i` 옵션을 사용하면 파일 하나하나를 삭제하기 전에 확인을 요청합니다.

```bash
$ rm -i important.txt
rm: remove regular file 'important.txt'? y
```

공유 디렉토리에서 파일을 삭제하거나, 많은 파일을 정리할 때, 또는 처음 명령어를 배울 때 `rm -i`를 사용하세요.

### -f 옵션으로 강제 삭제

`-f` 옵션은 "force"를 의미합니다. 존재하지 않는 파일을 무시하고 확인을 요청하지 않습니다.

```bash
$ rm -f old-cache.txt
```

스크립트에서 파일이 이미 없어도 정리를 계속해야 할 때 유용합니다.

```bash
$ rm -f build.log
```

주의하세요: `-f`는 일부 안전 확인 메시지도 숨기므로 실수를 감출 수 있습니다.

### -r 옵션으로 디렉토리 삭제

기본적으로 `rm`은 디렉토리를 삭제할 수 없습니다.

```bash
$ rm projects
rm: cannot remove 'projects': Is a directory
```

디렉토리와 그 안의 모든 내용을 삭제하려면 재귀적 삭제를 의미하는 `-r` 또는 `-R` 옵션을 사용하세요.

```bash
$ rm -r old-project
```

재귀적 삭제는 디렉토리 트리를 따라가며 파일, 하위 디렉토리 및 그 내용을 모두 삭제합니다.

### rm -rf의 위험성

`rm -rf` 명령은 재귀적 삭제와 강제 삭제를 결합한 것입니다.

```bash
$ rm -rf old-project
```

빌드 결과물 같은 생성된 폴더를 삭제할 때 적절할 수 있지만, 질문 없이 전체 트리를 삭제하기 때문에 위험합니다. 항상 다음을 확인하세요:

- 현재 위치가 예상한 디렉토리인지? `pwd`를 사용하세요.
- 와일드카드가 올바르게 확장되었는지? `ls`로 미리 확인하세요.
- 경로가 절대 경로인지 상대 경로인지? `/tmp/cache`와 `tmp/cache`는 매우 다릅니다.
- 실수로 공백이 있는지? `rm -rf old-project`와 `rm -rf old project`는 다른 경로를 대상으로 합니다.

### 빈 디렉토리는 rmdir 사용하기

더 안전한 대안으로 빈 디렉토리는 `rmdir`로 삭제할 수 있습니다.

```bash
$ rmdir empty-directory
```

`rmdir` 명령은 디렉토리가 완전히 비어 있을 때만 성공하므로, 정리 작업에서 `rm -r`보다 안전한 선택입니다.

### 자주 쓰이는 rm 옵션

가장 자주 보게 될 옵션은 다음과 같습니다:

- `-i`: 삭제하기 전에 매번 확인을 요청합니다.
- `-I`: 세 개 이상의 파일을 삭제하거나 재귀적으로 삭제할 때 한 번만 확인을 요청합니다.
- `-f`: 강제로 삭제하고 존재하지 않는 파일은 무시합니다.
- `-r` 또는 `-R`: 디렉토리와 그 내용을 재귀적으로 삭제합니다.
- `-v`: 삭제된 내용을 보여줍니다.

예를 들어, 옵션을 조합할 수도 있습니다:

```bash
$ rm -rv old-project
removed 'old-project/notes.txt'
removed directory 'old-project'
```

### 자주 묻는 질문

**rm을 실행 취소할 수 있나요?** 보통은 불가능합니다. `rm`으로 삭제한 파일은 내장된 실행 취소 명령이 없습니다. 백업, 버전 관리, 파일 시스템 복구 도구가 진짜 안전망입니다.

**rm이 "Permission denied"라고 하는 이유는?** 해당 파일을 삭제하거나 포함된 디렉토리를 수정할 권한이 없기 때문입니다. `ls -l`로 소유권과 권한을 확인하세요.

**rm이 "No such file or directory"라고 하는 이유는?** 해당 경로에 파일이 없거나 예상과 다른 디렉토리에 있기 때문입니다. `pwd`와 `ls`로 확인하세요.

**rm에 sudo를 사용해야 하나요?** 삭제하려는 경로를 완전히 이해할 때만 사용하세요. `sudo rm -r`은 일반 사용자 계정이 접근할 수 없는 시스템 파일도 삭제할 수 있습니다.
