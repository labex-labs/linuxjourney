## Lesson Content

파일 시스템 탐색법을 배운 후 다음 단계는 파일 내용을 보는 것입니다. 기본적이고 다재다능한 도구 중 하나가 `cat` 명령어입니다. `cat`은 "concatenate"의 줄임말로, 파일을 연결하는 기능을 암시합니다.

### 파일 내용 보기

`cat` 명령어의 가장 기본적인 용도는 단일 파일의 내용을 터미널에 직접 출력하는 것입니다.

```bash
$ cat myfile.txt
```

이 명령어는 `myfile.txt`의 전체 내용을 화면에 출력합니다. 짧은 설정 파일이나 텍스트 조각을 볼 때는 완벽하지만, 큰 파일을 볼 때는 텍스트가 너무 빨리 흘러가므로 적합하지 않습니다. 큰 파일에 더 적합한 도구는 이후 강의에서 다룰 예정입니다.

### 파일 연결하기

이름 그대로 `cat`은 여러 파일을 연결(concatenate)하여 결합된 출력을 보여줄 수 있습니다. 지정한 순서대로 파일을 읽고 순차적으로 출력합니다.

```bash
$ cat dogfile birdfile
```

이 명령어는 먼저 `dogfile`의 내용을 출력하고, 바로 이어서 `birdfile`의 내용을 출력합니다.

결합된 출력을 새 파일에 저장하려면 리디렉션을 사용하세요:

```bash
$ cat dogfile birdfile > animals
```

### 리디렉션으로 파일 생성하기

`cat`을 출력 리디렉션 연산자(`>`)와 함께 사용하여 새 파일을 만들 수도 있습니다. 터미널에서 직접 텍스트를 파일에 빠르게 쓸 수 있는 방법입니다.

```bash
$ cat > newfile.txt
```

이 명령어를 실행한 후 텍스트를 입력할 수 있습니다. 새 줄에서 `Ctrl+D`를 눌러 저장하고 종료하세요. 이렇게 하면 입력한 텍스트로 `newfile.txt`가 생성됩니다. 기존 파일에 `>`를 사용하면 완전히 덮어쓰므로 주의하세요.

덮어쓰지 않고 파일에 덧붙이려면 `>>`를 사용하세요.

```bash
$ cat >> notes.txt
```

### 자주 쓰이는 cat 명령어 옵션

`cat` 명령어에는 동작을 변경하는 여러 옵션이 있습니다.

- `-n`: 모든 출력 줄에 1부터 번호를 매깁니다.
- `-b`: 빈 줄을 제외한 출력 줄에만 번호를 매깁니다.
- `-s`: 여러 개의 빈 줄을 하나의 빈 줄로 압축합니다.
- `-A`: 출력하지 않는 문자, 탭, 줄 끝 문자를 표시합니다.

예시:

```bash
$ cat -n script.sh
$ cat -b notes.txt
$ cat -s messy.txt
```

### cat을 사용하지 말아야 할 때

짧은 파일에는 `cat`을 사용하세요. 긴 파일은 `less`를 사용해 스크롤, 검색, 종료가 가능하며 터미널이 넘치지 않습니다.

```bash
$ less /var/log/syslog
```

### 자주 묻는 질문

**cat은 무엇의 약자인가요?** concatenate의 약자입니다.

**cat으로 파일을 편집할 수 있나요?** 대화형 편집은 불가능합니다. 리디렉션으로 파일을 생성하거나 덮어쓸 수는 있지만, 편집에는 텍스트 편집기가 더 적합합니다.

**`>`와 `>>`의 차이는 무엇인가요?** `>`는 파일을 덮어씁니다. `>>`는 파일 끝에 내용을 덧붙입니다.
