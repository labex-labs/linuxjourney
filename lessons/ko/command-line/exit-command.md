## Lesson Content

리눅스 여정의 기초 강의를 완료하신 것을 축하합니다! 필수적인 리눅스 기본 사항을 익히셨고, 이제 세션을 올바르게 종료하는 방법을 배울 차례입니다. 리눅스 셸을 종료하는 것은 간단하지만 중요한 마지막 단계입니다.

### 종료 명령어

셸 세션을 종료하는 가장 일반적인 방법은 `exit` 명령어를 사용하는 것입니다. `exit`를 입력하고 Enter 키를 누르면 현재 셸 프로세스가 종료됩니다. 이 명령어는 거의 모든 셸 환경에서 작동합니다.

```bash
$ exit
```

터미널 창을 열었다면, `exit`는 보통 그 안에서 실행 중인 셸을 닫습니다. SSH로 연결한 경우 `exit`는 원격 셸 세션을 종료하고 로컬 프롬프트로 돌아갑니다.

### 종료 상태 값

`exit` 명령어는 상태 코드도 반환할 수 있습니다. 상태 코드가 `0`이면 보통 성공을 의미하고, 0이 아닌 값은 오류나 특별한 조건을 나타냅니다.

```bash
$ exit 0
```

종료 상태 코드는 셸 스크립트를 작성할 때 더 자주 보게 됩니다. 대화형 사용에서는 단순히 `exit`만 입력해도 충분합니다.

### 로그아웃 명령어

터미널 종료를 위해 사용할 수 있는 또 다른 명령어는 `logout`입니다. 이 명령어는 로그인 셸을 종료하도록 특별히 설계되었습니다. 많은 최신 시스템에서는 `exit`와 비슷하게 작동하지만, 두 명령어 모두 아는 것이 좋습니다.

```bash
$ logout
```

### 터미널 창 닫기

그래픽 사용자 인터페이스에서 작업 중이라면 터미널 창을 그냥 닫는 방법도 있습니다. 이 동작은 보통 그 안에서 실행 중인 셸 세션을 종료하는 신호를 보냅니다.

### 셸을 종료하는 일반적인 방법

- 현재 셸을 종료하려면 `exit`를 입력하세요.
- 빈 프롬프트에서 `Ctrl-D`를 눌러 EOF(파일 끝) 신호를 보내면 보통 셸이 종료됩니다.
- 로그인 셸에서는 `logout`을 입력하세요.
- 그래픽 터미널을 사용할 때는 터미널 창을 닫으세요.

### 자주 묻는 질문

**exit는 터미널 창을 닫는 것과 같은가요?** 결과는 비슷한 경우가 많지만, `exit`는 셸에 종료를 명확히 알리는 명령입니다.

**Ctrl-D는 무엇인가요?** 셸에 EOF 신호를 보냅니다. 빈 프롬프트에서 이 신호는 보통 셸을 종료합니다.

**exit 1은 무슨 뜻인가요?** 상태 코드 `1`로 종료하는 것으로, 스크립트에서 실패를 나타낼 때 흔히 사용됩니다.

이제 셸을 탐색하고, 파일을 다루고, 도움을 받고, 셸을 종료하는 방법을 성공적으로 배웠습니다.
