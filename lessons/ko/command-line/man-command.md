## Lesson Content

리눅스에서는 거의 모든 명령어가 자체 설명서와 함께 제공됩니다. 이를 "man 페이지"(manual pages의 줄임말)라고 하며, 시스템을 효과적으로 사용하는 방법을 배우는 데 필수적인 자료입니다.

### 매뉴얼 페이지 이해하기

man 페이지는 리눅스 명령어, 유틸리티, 시스템 호출에 대한 내장 문서입니다. 명령어가 무엇을 하는지, 사용 가능한 옵션(또는 플래그), 그리고 사용 방법을 자세히 설명합니다. 명령줄 도움말을 얻는 가장 첫 번째이자 최고의 출처입니다.

### man으로 매뉴얼 접근하기

어떤 명령어의 매뉴얼을 보려면 `man` 다음에 명령어 이름을 입력하세요. 예를 들어 `ls` 매뉴얼을 읽으려면 다음과 같이 입력합니다:

```bash
$ man ls
```

이렇게 하면 `ls` man 페이지가 열립니다. 화살표 키로 스크롤하고, `/`로 검색하며, `q`를 눌러 종료할 수 있습니다.

### 명령어 옵션 상세 정보 찾기

man 페이지는 명령어 옵션을 이해하는 데 특히 유용합니다. 예를 들어 `ls -l`을 본 적이 있고 `-l`이 무엇을 의미하는지 알고 싶다면 `man ls`를 열고 `-l`을 검색하세요.

man 페이지 내에서:

- `/`를 누르고 검색어를 입력하여 앞으로 검색합니다.
- `n`을 눌러 다음 검색 결과로 이동합니다.
- `N`을 눌러 이전 검색 결과로 이동합니다.
- `q`를 눌러 종료합니다.

### man 페이지 섹션 이해하기

매뉴얼 페이지는 번호가 매겨진 섹션으로 구성되어 있습니다. 일반적인 섹션은 다음과 같습니다:

- `1`: 사용자 명령어.
- `2`: 시스템 호출.
- `3`: 라이브러리 함수.
- `5`: 파일 형식.
- `8`: 시스템 관리 명령어.

때때로 같은 이름이 여러 섹션에 존재할 수 있습니다. 이 경우 섹션 번호를 지정할 수 있습니다:

```bash
$ man 5 passwd
$ man 1 passwd
```

### 자주 묻는 질문

**왜 man 출력이 이렇게 길까요?** man 페이지는 참고용 문서입니다. 필요한 부분으로 바로 가려면 man 내 검색 기능을 사용하세요.

**man을 종료하려면 어떻게 하나요?** `q`를 누르세요.

**만약 man 페이지가 없다면?** `COMMAND --help`, `help COMMAND`를 시도하거나 배포판의 문서 패키지를 설치해 보세요.
