## Lesson Content

### 리눅스 쉘이란 무엇인가

리눅스 여정에 오신 것을 환영합니다! 첫 번째 단계는 리눅스 쉘을 이해하는 것입니다. 쉘은 사용자가 입력한 명령어를 받아 운영체제에 실행을 요청하고, 그 결과를 터미널에 출력하는 프로그램입니다.

그래픽 사용자 인터페이스를 사용해본 적이 있다면, 창, 메뉴, 버튼을 클릭하는 데 익숙할 것입니다. 커맨드 라인에서는 대신 정확한 명령어를 입력합니다. 보통 "Terminal", "Console", 또는 "Konsole"이라는 이름의 애플리케이션이 쉘 세션을 열어줍니다.

쉘은 빠르고, 스크립트 작성이 가능하며, 거의 모든 리눅스 시스템에서 사용할 수 있기 때문에 유용합니다. 더 많은 명령어를 배우면서 파일 검사, 디렉터리 관리, 텍스트 검색, 소프트웨어 설치, 반복 작업 자동화 등을 조합하여 수행할 수 있습니다.

### Bash 쉘과 상호작용하기

이 강의에서는 Bash, 즉 Bourne Again Shell에 집중할 것입니다. Bash는 가장 흔한 리눅스 쉘 중 하나이며, 나중에 `zsh`, `fish` 또는 다른 쉘을 사용하더라도 좋은 기초가 됩니다.

터미널을 열면 쉘 프롬프트가 나타납니다. 모양은 다양할 수 있지만, 보통 사용자 이름, 호스트 이름, 현재 디렉터리를 표시합니다.

```plaintext
pete@icebox:/home/pete $
```

`$` 기호는 쉘이 일반 사용자로서 입력을 받을 준비가 되었음을 나타냅니다. 명령어를 입력할 때 이 기호는 치지 않습니다; 쉘이 보여주는 표시입니다. 만약 `#`가 보인다면, 보통 루트 사용자로 작업 중이며 더 많은 권한과 위험이 있습니다.

명령어는 보통 다음과 같은 형태를 따릅니다:

```bash
command options arguments
```

예를 들어, `echo Hello World`에서 `echo`는 명령어이고 `Hello World`는 명령어에 전달되는 텍스트입니다.

### 당신의 첫 번째 리눅스 명령어

초보자를 위한 가장 기본적인 리눅스 명령어 중 하나인 `echo`부터 시작해봅시다. 이 명령어는 입력한 텍스트를 터미널에 다시 출력합니다.

```bash
$ echo Hello World
Hello World
```

몇 가지 예제를 더 시도해보세요:

```bash
$ echo Linux is fun
Linux is fun
$ echo "Hello from Bash"
Hello from Bash
```

쉘이 여러 단어를 하나의 텍스트 덩어리로 처리하게 하려면 따옴표를 사용하는 것이 유용합니다.

### 초보자를 위한 일반 팁

- 명령어를 실행하려면 `Enter` 키를 누르세요.
- 이전 명령어를 불러오려면 `위쪽 화살표` 키를 사용하세요.
- 리눅스에서는 명령어와 파일 이름이 대소문자를 구분합니다.
- 공백이 중요합니다. `echo hello`와 `echohello`는 다릅니다.
- 명령어가 멈춘 것 같으면 `Ctrl-C`를 눌러 취소할 수 있습니다.

### 자주 묻는 질문

**쉘과 터미널은 같은 것인가요?** 정확히는 아닙니다. 터미널은 입력하는 창이나 앱이고, 쉘은 그 안에서 실행되는 프로그램입니다.

**예제에 나오는 `$`를 직접 입력해야 하나요?** 아닙니다. `$`는 프롬프트 표시일 뿐입니다. 그 뒤에 나오는 명령어만 입력하세요.

**다른 쉘도 있는데 왜 Bash를 배워야 하나요?** Bash는 널리 사용되고 문서화가 잘 되어 있으며, 튜토리얼과 스크립트에서 흔히 사용됩니다.
