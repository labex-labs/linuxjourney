## Lesson Content

Linux 커맨드 라인에서 작업할 때, 명령어가 어떻게 작동하는지 또는 어떤 옵션을 받는지 빠르게 확인해야 할 때가 많습니다. Linux는 터미널 내에서 직접 여러 가지 도움말 도구를 제공합니다.

### Bash 내장 명령어를 위한 'help' 명령어

가장 직접적인 도구 중 하나는 `help`입니다. 이 명령어는 Bash 셸에 내장되어 있으며, 다른 Bash 내장 명령어에 대한 정보를 제공하도록 특별히 설계되었습니다. 내장 명령어는 별도의 프로그램이 아니라 셸 자체의 일부입니다. 예를 들어 `echo`, `cd`, `pwd` 등이 있습니다.

`help`를 사용하려면 내장 명령어 이름 뒤에 입력하세요.

```bash
$ help echo
```

이 명령어는 `echo` 명령어의 요약, 구문, 그리고 사용 가능한 옵션 목록을 보여줍니다. 셸 전용 기능에 대한 도움을 가장 빠르게 얻는 방법입니다.

### 실행 가능한 프로그램을 위한 --help 플래그

셸에 내장되지 않은 대부분의 실행 가능한 프로그램에는 `help` 명령어가 작동하지 않습니다. 대신, 일반적인 관례로 `--help` 플래그를 제공합니다. 이 옵션은 프로그램에 사용법 요약을 출력하고 종료하도록 지시합니다.

```bash
$ ls --help
```

대부분의 개발자가 이 표준을 따르지만, 모든 프로그램에 해당하는 것은 아닙니다. 낯선 프로그램에 대해선 `--help`를 시도하는 것이 보통 좋은 첫걸음입니다.

### 명령어 유형 찾기

명령어가 Bash 내장인지 외부 프로그램인지 확실하지 않을 때는 `type`을 사용하세요.

```bash
$ type cd
cd is a shell builtin
$ type ls
ls is /usr/bin/ls
```

이 명령어는 `help cd`, `ls --help`, 또는 `man ls` 중 어떤 도움말을 사용할지 결정하는 데 도움을 줍니다.

### 적절한 도움말 도구 선택하기

- `cd`, `echo`, `history` 같은 Bash 내장 명령어에는 `help COMMAND`를 사용하세요.
- 많은 외부 명령어에는 빠른 요약을 위해 `COMMAND --help`를 사용하세요.
- 자세한 매뉴얼 페이지가 필요하면 `man COMMAND`를 사용하세요.
- 한 줄 설명이 필요하면 `whatis COMMAND`를 사용하세요.

### 자주 묻는 질문

**왜 help ls는 작동하지 않나요?** `ls`는 보통 Bash 내장이 아닌 외부 프로그램입니다. `ls --help` 또는 `man ls`를 시도하세요.

**왜 모든 명령어에 --help가 작동하지 않나요?** 이것은 관례일 뿐 엄격한 규칙이 아닙니다.

**명령어를 가장 빠르게 확인하는 방법은 무엇인가요?** 외부 명령어는 `COMMAND --help`, Bash 내장 명령어는 `help COMMAND`를 시도하세요.
