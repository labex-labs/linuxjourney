## Lesson Content

システム上に無数のファイルがあると、特定のファイルを見つけるのは難しいことがあります。`find` コマンドは、名前、タイプ、サイズ、変更時間などの条件を使ってディレクトリツリーを検索します。

### find コマンドの使い方

基本的な構文は以下の通りです：

```bash
find [PATH] [EXPRESSION]
```

検索するディレクトリと探したい条件を指定します。

例えば、`/home` ディレクトリとそのすべてのサブディレクトリ内で `puppies.jpg` という名前のファイルを検索するには、次のようにします：

```bash
$ find /home -name puppies.jpg
```

検索はデフォルトで再帰的に行われるため、`find /home` は `/home` とそのサブディレクトリ内を検索します。

### 名前とタイプで検索する

`find` の最も一般的な使い方の一つは、ファイル名での検索です。`-name` オプションは名前を正確に、またはシェルスタイルのパターンで一致させます。

```bash
$ find . -name "*.txt"
```

検索対象のアイテムのタイプも指定できます。`-type` オプションを使います。例えば、ファイルではなくディレクトリを探したい場合は `d` を使います。

```bash
$ find /home -type d -name MyFolder
```

このコマンドではタイプを `d`（ディレクトリ）に設定し、`MyFolder` という名前のアイテムを検索しています。通常のファイルを指定する場合は `-type f` を使います。

### サイズと時間で検索する

ファイルサイズで検索できます：

```bash
$ find . -type f -size +10M
$ find . -type f -size -1k
```

最初のコマンドは10メガバイトより大きいファイルを検索し、2つ目は1キロバイトより小さいファイルを検索します。

変更時間で検索も可能です：

```bash
$ find . -type f -mtime -7
$ find . -type f -mtime +30
```

`-mtime -7` は過去7日以内に変更されたファイルを意味し、`-mtime +30` は30日以上前に変更されたファイルを意味します。

### 検索結果に対して操作を実行する

デフォルトでは、`find` は一致したパスを表示します。`-print`、`-delete`、`-exec` などの操作を追加できます。

一致した結果を明示的に表示する：

```bash
$ find . -name "*.log" -print
```

一致した各ファイルに対して `ls -l` を実行する：

```bash
$ find . -name "*.log" -exec ls -l {} \;
```

`{}` は一致したパスに置き換えられます。エスケープされたセミコロンはコマンドの終了を示します。

`-delete` のような破壊的な操作は注意が必要です。まずは `-delete` を付けずに同じ検索を実行して結果を確認してください。

### よく使う find オプション

- `-name PATTERN`：ファイル名で一致させる。
- `-iname PATTERN`：大文字小文字を無視してファイル名で一致させる。
- `-type f`：通常のファイルに一致させる。
- `-type d`：ディレクトリに一致させる。
- `-size +10M`：10メガバイトより大きいファイルに一致させる。
- `-mtime -7`：過去7日以内に変更されたファイルに一致させる。
- `-maxdepth N`：`find` の検索深度を制限する。

### よくある質問

**なぜ find で「Permission denied」と表示されるのですか？** ユーザーにそのディレクトリを読む権限がありません。より狭いパスを検索するか、適切な権限で実行してください。

**なぜ "*.txt" のようなパターンを引用符で囲む必要があるのですか？** 引用符で囲むことで、シェルがワイルドカードを展開せずに `find` に渡すことができます。

**find は再帰的ですか？** はい。デフォルトでサブディレクトリも検索します。
