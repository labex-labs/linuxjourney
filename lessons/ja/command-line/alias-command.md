## Lesson Content

長くて繰り返し入力するコマンドは面倒です。エイリアスは、コマンドやコマンドの連続に対してカスタム名を定義できるシェルのショートカットです。

### 一時的なエイリアスの作成

現在のターミナルセッションでのみ有効な一時的なエイリアスを作成するには、名前を指定し、それをコマンド文字列に等しく設定します。

例えば、`ls -la` のエイリアスとして `ll` を作成します。

```bash
$ alias ll='ls -la'
```

これで、`ls -la` と入力する代わりに `ll` と入力するだけで、同じコマンドが実行されます。これはシェルをカスタマイズするシンプルで強力な方法です。

### エイリアスを永続化する

一時的なエイリアスは、ターミナルを閉じるかシステムを再起動すると消えてしまいます。Linuxでコマンドエイリアスを永続化するには、シェルの設定ファイルに追加する必要があります。Bashシェルの場合、このファイルは通常 `~/.bashrc` です。

1. テキストエディタでファイルを開きます：`nano ~/.bashrc`
2. コマンドラインで入力したのと同じようにエイリアス定義をファイルに追加します。

```bash
alias ll='ls -la'
alias update='sudo apt update && sudo apt upgrade'
```

3. ファイルを保存してエディタを終了します。

変更を反映させるには、ターミナルを閉じて再度開くか、`source` コマンドで設定ファイルを再読み込みします。

```bash
$ source ~/.bashrc
```

これで、新しいBashセッションを開始するたびにエイリアスが利用可能になります。

### エイリアスの削除

不要になったエイリアスは、`unalias` コマンドで削除できます。これは現在のセッションから削除します。

```bash
$ unalias ll
```

永続的なエイリアスを削除するには、`~/.bashrc` ファイルからも定義を削除する必要があります。

### 既存のエイリアスの一覧表示

引数なしで `alias` を実行すると、現在のシェルのエイリアス一覧が表示されます。

```bash
$ alias
alias ll='ls -la'
alias grep='grep --color=auto'
```

`type` コマンドを使うと、コマンド入力時に何が実行されるか確認できます。

```bash
$ type ll
ll is aliased to 'ls -la'
```

### 便利なエイリアス例

```bash
$ alias la='ls -la'
$ alias ..='cd ..'
$ alias grep='grep --color=auto'
```

エイリアスは短く予測可能に保ちましょう。破壊的なコマンドを予期しない動作に置き換えるのは、十分に確信がある場合を除き避けてください。

### よくある質問

**エイリアスはスクリプトで使えますか？** 通常は使えません。スクリプトでは完全なコマンドや関数を使うべきです。

**Bashのエイリアスはどこに置くべきですか？** 対話型Bashセッション用には `~/.bashrc` に置きます。

**エイリアスが実際のコマンドと競合したら？** 対話型シェルでは通常エイリアスが優先されます。エイリアスを回避するには `command name` または `\name` を使います。
