## Lesson Content

Linuxでは、ファイルやディレクトリはファイルシステムと呼ばれる階層構造で整理されています。自信を持って移動する前に、自分がどこにいるのかを知る必要があります。`pwd`コマンドは、現在の作業ディレクトリを表示することでその疑問に答えます。

### The Directory Tree in Linux

ファイルシステム全体は、ルートディレクトリと呼ばれる単一の最上位ディレクトリから始まります。これはスラッシュ（`/`）で表されます。ルートからディレクトリツリーはサブディレクトリに分岐し、それらはファイルやさらに多くのサブディレクトリを含むことができます。

以下はこの構造の簡略化した例です：

```plaintext
/
|-- bin
|   |-- file1
|   |-- file2
|-- etc
|   |-- file3
|   `-- directory1
|       |-- file4
|       `-- file5
|-- home
|-- var
```

### Understanding File Paths

任意のファイルやディレクトリの場所は、そのパスで表されます。パスとは、ある出発点から特定の目的地まで導くディレクトリの連なりです。

例えば、`/home`の中に`pete`というフォルダがあり、その中に`Movies`フォルダがある場合、完全なパスは次のようになります：

```plaintext
/home/pete/Movies
```

`/`で始まるパスはルートディレクトリから始まるため絶対パスです。`Movies`のようなパスは現在の場所に依存するため相対パスです。

### What is the Full Form of PWD in Linux?

`pwd`の正式名称は「print working directory（作業ディレクトリの表示）」です。作業ディレクトリとは、シェルが現在いるディレクトリのことです。相対パスを使うコマンドはこの場所から始まります。

### Using the pwd Command

現在のディレクトリを知るには、`pwd`と入力してEnterキーを押します。

```bash
$ pwd
/home/pete
```

出力は絶対パスです。この例では、シェルは現在`pete`ユーザーのホームディレクトリにいます。

### Why pwd is Useful

`pwd`を使うのは以下の場合です：

- 指示に従っていて自分の場所を確認したいとき。
- ファイルパスが間違っていてコマンドが失敗したとき。
- いくつかのディレクトリを移動してどこにいるか分からなくなったとき。
- 現在のディレクトリパスを別のコマンドにコピーしたいとき。

例えば：

```bash
$ pwd
/home/pete/projects
$ ls
app.py  README.md
```

これは`app.py`と`README.md`が`/home/pete/projects`にあることを示しています。

### Common Questions

**pwdは何かを変更しますか？** いいえ。`pwd`は情報を表示するだけです。

**なぜ私のシステムで出力が違うのですか？** ユーザー名やホームディレクトリ、現在の場所が異なるためです。

**pwdとcdの違いは何ですか？** `pwd`は現在地を表示します。`cd`は現在地を変更します。
