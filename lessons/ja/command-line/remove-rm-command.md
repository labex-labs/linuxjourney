## Lesson Content

Linuxでは、不要になったファイルがたまることがよくあります。それらを削除するには、ファイルシステム管理の基本的なユーティリティである`rm`（remove）コマンドを使います。基本的な構文は以下の通りです。

```bash
rm [OPTIONS] FILE...
```

`rm`コマンドはファイルシステムからディレクトリエントリを削除します。つまり、ファイルを削除するということです。多くのデスクトップ環境とは異なり、コマンドラインでの削除は通常ファイルをゴミ箱に移動しないため、Enterキーを押す前にコマンドをよく確認してください。

### 単一ファイルの削除

1つのファイルを削除するには、ファイル名を`rm`に渡します。

```bash
$ rm file1
```

複数のファイルを一度に削除するには、ファイル名を続けて指定します。

```bash
$ rm notes.txt old-report.txt draft.md
```

これは素早い整理に便利ですが、タイプミスで意図しないファイルまで削除してしまうこともあります。

### ワイルドカードを使ったファイル削除

シェルのワイルドカードを使うと複数のファイルをマッチさせることができます。例えば、現在のディレクトリ内のすべての`.tmp`ファイルを削除するには以下のようにします。

```bash
$ rm *.tmp
```

`rm`をワイルドカードと使う前に、`ls`でマッチするファイルを確認するほうが安全です。

```bash
$ ls *.tmp
cache.tmp  test.tmp
$ rm *.tmp
```

シェルは`rm`が実行される前に`*.tmp`を展開します。パターンが予想以上に多くのファイルにマッチした場合でも、`rm`はすべてのファイルを受け取ります。

### -iオプションで対話的削除

より安全に削除するには、`-i`オプションを使います。これは削除前にファイルごとに確認を求めます。

```bash
$ rm -i important.txt
rm: remove regular file 'important.txt'? y
```

共有ディレクトリからの削除や大量のファイル整理、コマンドを初めて使うときに`rm -i`を使うと良いでしょう。

### -fオプションで強制削除

`-f`オプションは「強制」を意味します。存在しないファイルを無視し、確認も求めません。

```bash
$ rm -f old-cache.txt
```

これはスクリプト内で、ファイルが既に無くても処理を続けたい場合に便利です。

```bash
$ rm -f build.log
```

注意してください：`-f`は安全確認のプロンプトも抑制するため、誤操作を隠してしまうことがあります。

### -rオプションでディレクトリ削除

デフォルトでは、`rm`はディレクトリを削除できません。

```bash
$ rm projects
rm: cannot remove 'projects': Is a directory
```

ディレクトリとその中身すべてを削除するには、再帰的削除のために`-r`または`-R`を使います。

```bash
$ rm -r old-project
```

再帰的削除はディレクトリツリーを辿り、ファイルやサブディレクトリ、その中身をすべて削除します。

### rm -rfの危険性

`rm -rf`コマンドは再帰的削除と強制削除を組み合わせたものです。

```bash
$ rm -rf old-project
```

ビルド成果物など生成されたフォルダを削除するのに適切な場合もありますが、質問なしにツリー全体を削除するため非常に危険です。必ず以下を確認してください：

- 自分がいるディレクトリは正しいか？`pwd`で確認。
- ワイルドカードの展開は正しいか？`ls`でプレビュー。
- パスは絶対パスか相対パスか？`/tmp/cache`と`tmp/cache`は全く違います。
- 誤ってスペースが入っていないか？`rm -rf old-project`と`rm -rf old project`は別のパスを指定します。

### 空のディレクトリを削除するにはrmdirを使う

より安全な方法として、空のディレクトリは`rmdir`で削除できます。

```bash
$ rmdir empty-directory
```

`rmdir`はディレクトリが完全に空の場合のみ成功するため、`rm -r`より安全な選択肢です。

### よく使うrmオプション

よく使うオプションは以下の通りです：

- `-i`：削除前に毎回確認する。
- `-I`：3つ以上のファイルや再帰削除時に一度だけ確認する。
- `-f`：強制削除し、存在しないファイルは無視する。
- `-r`または`-R`：ディレクトリとその中身を再帰的に削除する。
- `-v`：削除したファイルを表示する。

例えば、オプションを組み合わせることもできます。

```bash
$ rm -rv old-project
removed 'old-project/notes.txt'
removed directory 'old-project'
```

### よくある質問

**rmを元に戻せますか？** 通常はできません。`rm`で削除したファイルを元に戻す組み込みコマンドはありません。バックアップやバージョン管理、ファイルシステムの復旧ツールが本当の安全策です。

**なぜrmで「Permission denied」と表示されるのですか？** ファイルの削除やその親ディレクトリの変更権限がありません。`ls -l`で所有者と権限を確認してください。

**なぜrmで「No such file or directory」と表示されるのですか？** 指定したパスにファイルが存在しないか、想定と違うディレクトリにいる可能性があります。`pwd`や`ls`で確認しましょう。

**rmにsudoを使うべきですか？** 削除するパスを完全に理解している場合のみ使ってください。`sudo rm -r`は通常のユーザー権限では触れないシステムファイルを削除する可能性があります。
