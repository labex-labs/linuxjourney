## Lesson Content

`cp`コマンドはLinuxでファイルやディレクトリをコピーする標準的なツールです。元のファイルはそのままにして、新しいコピーを作成します。基本的な構文は以下の通りです。

```bash
cp [OPTIONS] SOURCE DESTINATION
```

1つのファイルを別のファイルにコピーしたり、1つ以上のファイルをディレクトリにコピーしたり、適切なオプションを使うことでディレクトリツリー全体をコピーすることもできます。

### 基本的なファイルコピー

ファイルをコピーするには、コピー元のファイルとコピー先のディレクトリまたはパスを指定します。

```bash
$ cp mycoolfile /home/pete/Documents/cooldocs
```

この例では、`mycoolfile`がコピー元のファイルで、`/home/pete/Documents/cooldocs`がコピー先のディレクトリです。コピー先でファイル名を変えてコピーすることもできます。

```bash
$ cp mycoolfile /home/pete/Documents/mycoolfile_backup
```

コピー先が既存のディレクトリの場合、コピーされたファイルは元の名前を保持します。コピー先がファイル名の場合、`cp`はその新しい名前でコピーを作成します。

### 複数ファイルをディレクトリにコピーする

複数のファイルを同じディレクトリにコピーするには、すべてのコピー元を先に列挙し、最後にコピー先のディレクトリを指定します。

```bash
$ cp report.txt notes.txt summary.txt /home/pete/Documents/
```

複数のコピー元を指定する場合、最後の引数は必ずディレクトリでなければなりません。

### ワイルドカードを使った一括コピー

ワイルドカードはパターンに基づいて複数のファイルを選択するための特殊な文字で、非常に柔軟に使えます。

- `*`：任意の文字列にマッチします。
- `?`：任意の1文字にマッチします。
- `[]`：括弧内のいずれか1文字にマッチします。

例えば、現在の場所からすべてのJPEG画像を`Pictures`ディレクトリにコピーするには：

```bash
$ cp *.jpg /home/pete/Pictures
```

コピー前にマッチするファイルを確認することもできます：

```bash
$ ls *.jpg
beach.jpg  lunch.jpg  profile.jpg
$ cp *.jpg /home/pete/Pictures
```

### ディレクトリを再帰的にコピーする

オプションなしで`cp`を使ってディレクトリをコピーしようとするとエラーになります。ディレクトリとその中身（サブディレクトリを含む）をコピーするには、`-r`（再帰的）フラグを使う必要があります。

```bash
$ cp -r Pumpkin/ /home/pete/Documents
```

このコマンドは`Pumpkin`ディレクトリとその中身すべてを`Documents`ディレクトリにコピーします。

一般的なLinuxシステムでは、同じ再帰的な目的で`-R`オプションも使われます：

```bash
$ cp -R website /home/pete/backups/
```

### ファイルの上書き処理

デフォルトでは、コピー先に同名のファイルがある場合、`cp`は上書きします。誤ってデータを失わないように、上書き前に確認を求める`-i`（インタラクティブ）フラグを使うことができます。

```bash
$ cp -i mycoolfile /home/pete/Pictures
cp: overwrite '/home/pete/Pictures/mycoolfile'? n
```

逆に、プロンプトなしで強制的に上書きしたい場合は`-f`オプションを使います。スクリプトなどでユーザー操作ができない場合に便利です。

```bash
$ cp -f mycoolfile /home/pete/Pictures
```

もう一つ便利な安全オプションは`-n`で、「上書きしない」という意味です。既存のコピー先ファイルを上書きしません。

```bash
$ cp -n mycoolfile /home/pete/Pictures
```

### -pでファイル属性を保持する

ファイルをコピーすると、通常は修正日時や所有権などのメタデータが更新されます。これらの元の属性を保持したい場合は、`-p`オプションを使います。

`cp -p`オプションはバックアップやタイムスタンプを保持したいファイル移行時に特に役立ちます。

```bash
$ cp -p mycoolfile /home/pete/backups/
```

これは`mycoolfile`をモード、許可されている所有権、タイムスタンプを保持してコピーします。

### -aでアーカイブコピー

`-a`オプションはアーカイブを意味します。多くの属性を保持しつつ再帰的にコピーするため、バックアップスタイルのディレクトリコピーに一般的に使われます。

```bash
$ cp -a project/ project-backup/
```

日常的なバックアップでは、複数のオプションを組み合わせるより`cp -a`の方が便利です。

### -uで新しいファイルだけコピー

`-u`オプションは、コピー元のファイルがコピー先より新しい場合、またはコピー先ファイルが存在しない場合にのみコピーします。

```bash
$ cp -u *.txt /home/pete/Documents/
```

これは、すでに最新のファイルを上書きせずにフォルダを更新したいときに便利です。

### よく使うcpオプション

よく使うオプションは以下の通りです：

- `-r` または `-R`：ディレクトリを再帰的にコピー。
- `-i`：上書き前に確認を求める。
- `-f`：必要に応じてコピー先を削除して強制上書き。
- `-n`：既存ファイルを上書きしない。
- `-p`：モード、所有権（可能な範囲で）、タイムスタンプを保持。
- `-a`：アーカイブモード。ディレクトリツリーを保持してコピー。
- `-u`：コピー元が新しい場合のみコピー。
- `-v`：コピー中のファイルを表示。

### よくある質問

**なぜcpはファイルを上書きしたのですか？** デフォルトでは、`cp`は同名のコピー先ファイルを置き換えます。上書き前に確認したい場合は`cp -i`を、上書きを避けたい場合は`cp -n`を使いましょう。

**なぜcpはディレクトリをコピーできないのですか？** ディレクトリは再帰的コピーが必要です。`cp -r source-dir destination-dir`を使ってください。

**cpとmvの違いは何ですか？** `cp`はコピーを作成し元のファイルは残します。`mv`は元のファイルを移動または名前変更します。

**バックアップにはcp -rとcp -aどちらを使うべきですか？** 単純な再帰コピーなら`cp -r`を使います。より多くのファイル属性を保持したいバックアップには`cp -a`を使いましょう。
