## Lesson Content

シェルは、これまでに入力したコマンドの記録を保持しています。コマンドを再入力せずに見つけて再利用したいときに、このリストにアクセスできます。`history` コマンドは、Bashや多くのUnix系シェル環境で基本的なツールです。

### コマンド履歴の表示

使用したコマンドの一覧を見るには、`history` と入力します。

```bash
$ history
  101  pwd
  102  ls -la
  103  cat notes.txt
```

各行には履歴番号とコマンドが表示されます。

### 以前のコマンドの再実行

シェルはコマンドの再実行を簡単にするためのショートカットをいくつか提供しています。

- **上矢印キー**: 直前に実行したコマンドをもう一度実行したい場合は、上矢印キーを押して履歴を遡ります。
- **`!!` ショートカット**: 直近のコマンドをもう一度実行するには、`!!` を使います。例えば、`cat file1` を実行した直後に `!!` と入力してEnterを押すと、再び `cat file1` が実行されます。
- **番号で実行**: `!102` と入力すると、履歴番号102のコマンドを実行します。
- **接頭辞で実行**: `!cat` と入力すると、直近で `cat` で始まるコマンドを実行します。

### 履歴の検索

最も強力な履歴ショートカットの一つが `Ctrl-R` です。これを押すと逆方向検索が始まります。`Ctrl-R` を押した後、探しているコマンドの一部を入力すると、シェルは最新の一致を表示します。さらに `Ctrl-R` を繰り返し押すと、古い一致を順に表示します。目的のコマンドが見つかったら、Enterを押して実行できます。

実行前にコマンドを編集したい場合は、Enterの代わりに右矢印キーまたは左矢印キーを押します。

### 履歴リストの管理

履歴の表示だけでなく、直接管理することもできます。

- **現在の履歴リストをクリア**: `history -c` はメモリ上の履歴リストをすべて削除します。
- **履歴をファイルに書き込む**: `history -w` は現在のセッションの履歴を通常は `~/.bash_history` に保存します。
- **特定のエントリを削除**: `history -d <offset>` は履歴番号で指定したコマンドを一つ削除します。

例:

```bash
$ history -d 101
$ history -w
```

`!!` や `!102` のような履歴展開コマンドは注意して使いましょう。何が実行されるかを確認するために、まず `history` を使うことをおすすめします。

### その他の便利な端末ツール

端末の画面がいっぱいになったら、画面をきれいにしたくなるかもしれません。`clear` コマンドを使うと、表示を消して新しい画面から始められます。

```bash
$ clear
```

もう一つ欠かせない機能がタブ補完です。コマンド、ファイル名、ディレクトリの先頭を入力し、Tabキーを押すと、シェルが補完を試みます。複数の候補がある場合は、選択肢を表示したり何もしなかったりします。Tabキーをもう一度押すと、候補の一覧が表示されることが多いです。

### よくある質問

**Bashの履歴はどこに保存されていますか？** 通常は `~/.bash_history` に保存されますが、シェルの設定によって異なります。

**履歴はすぐにすべてのコマンドを含みますか？** そうとは限りません。多くのシェルはセッション終了時に履歴を書き込みますが、設定によっては異なります。

**履歴に機密情報が含まれることはありますか？** はい。パスワードやトークン、秘密情報を直接コマンドに入力するのは避けてください。

**`history -c` と `clear` の違いは何ですか？** `history -c` はメモリ上のコマンド履歴を消去します。`clear` は端末画面を消去するだけです。
