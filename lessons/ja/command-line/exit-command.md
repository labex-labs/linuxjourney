## Lesson Content

Linuxの基礎レッスンを修了おめでとうございます！基本的なLinuxの知識を身につけたので、次はセッションを正しく終了する方法を学びましょう。Linuxシェルを終了することは簡単ですが重要な最後のステップです。

### Exit コマンド

シェルセッションを終了する最も一般的な方法は `exit` コマンドです。`exit` と入力してEnterキーを押すと、現在のシェルプロセスが終了します。このコマンドはほぼすべてのシェル環境で動作します。

```bash
$ exit
```

ターミナルウィンドウを開いている場合、`exit` は通常その中で動作しているシェルを閉じます。SSHで接続している場合は、`exit` がリモートシェルセッションを終了し、ローカルのプロンプトに戻ります。

### 終了ステータスの値

`exit` コマンドは終了ステータスコードを返すこともできます。ステータスが `0` の場合は通常成功を意味し、0以外の値はエラーや特別な状態を示します。

```bash
$ exit 0
```

終了ステータスはシェルスクリプトを書く際によく使われます。対話的に使う場合は単に `exit` と入力するだけで十分です。

### Logout コマンド

ターミナルを終了するもう一つのコマンドは `logout` です。このコマンドはログインシェルを終了するために特化しています。多くの現代的なシステムでは `exit` と似た動作をしますが、両方のコマンドを知っておくことは良い習慣です。

```bash
$ logout
```

### ターミナルウィンドウを閉じる

グラフィカルユーザーインターフェースを使っている場合は、単にターミナルウィンドウを閉じることもできます。この操作は通常、内部で動作しているシェルセッションを終了させる信号を送ります。

### シェルを終了する一般的な方法

- 現在のシェルを終了するには `exit` と入力する。
- 空のプロンプトで `Ctrl-D` を押すと、EOF（ファイルの終わり）を送信し、多くの場合シェルが終了する。
- ログインシェルの場合は `logout` と入力する。
- グラフィカルターミナルを使っている場合はターミナルウィンドウを閉じる。

### よくある質問

**exitはターミナルウィンドウを閉じるのと同じですか？** 結果は似ていることが多いですが、`exit` はシェルに終了を明確に伝えます。

**Ctrl-Dとは何ですか？** シェルにEOF信号を送ります。空のプロンプトでこれを押すと通常シェルが終了します。

**exit 1とは何ですか？** ステータスコード `1` で終了します。スクリプトで失敗を示すためによく使われます。
