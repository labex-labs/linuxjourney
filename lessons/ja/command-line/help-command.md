## Lesson Content

Linuxのコマンドラインで作業していると、コマンドの使い方や受け付けるオプションをすぐに確認したくなることがよくあります。Linuxはターミナル内で直接使えるいくつかのヘルプツールを提供しています。

### Bash組み込みコマンド用の 'help' コマンド

最も直接的なツールの一つが `help` です。これはBashシェルに直接組み込まれているコマンドで、他のBash組み込みコマンドに関する情報を提供するために特化しています。組み込みコマンドとは、シェル自体の一部であり、別のプログラムではありません。例としては `echo`、`cd`、`pwd` などがあります。

`help` を使うには、その後に組み込みコマンドの名前を続けて入力します。

```bash
$ help echo
```

これにより、`echo` コマンドの概要、構文、利用可能なオプションの一覧が表示されます。シェル固有の機能について素早く助けを得る最速の方法です。

### 実行可能プログラム用の --help フラグ

シェルに組み込まれていないほとんどの実行可能プログラムに対しては、`help` コマンドは機能しません。その代わりに、`--help` フラグを提供するのが一般的な慣習です。このオプションはプログラムに使用法の概要を表示して終了するよう指示します。

```bash
$ ls --help
```

ほとんどの開発者はこの標準に従っていますが、必ずしも全てではありません。慣れないプログラムにはまず `--help` を試すのが良いでしょう。

### コマンドの種類を調べる

コマンドがBashの組み込みか外部プログラムか分からない場合は、`type` を使います。

```bash
$ type cd
cd is a shell builtin
$ type ls
ls is /usr/bin/ls
```

これにより、`help cd`、`ls --help`、または `man ls` のどれを使うべきか判断できます。

### 適切なヘルプツールの選び方

- `cd`、`echo`、`history` のようなBash組み込みには `help COMMAND` を使う。
- 多くの外部コマンドには `COMMAND --help` で簡単な概要を得る。
- 詳細なマニュアルページは `man COMMAND` を使う。
- 一行説明が欲しい場合は `whatis COMMAND` を使う。

### よくある質問

**なぜ help ls は動かないのですか？** `ls` は通常Bashの組み込みではなく外部プログラムだからです。`ls --help` や `man ls` を試してください。

**なぜすべてのコマンドで --help が動くわけではないのですか？** これは慣習であり、厳密なルールではありません。

**コマンドを素早く確認する最速の方法は？** 外部コマンドには `COMMAND --help`、Bash組み込みには `help COMMAND` を試してください。
